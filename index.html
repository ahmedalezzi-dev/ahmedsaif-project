Dataset Overview

This dataset focuses on air quality assessment across various regions, capturing critical environmental and demographic factors that influence pollution levels.

Key Features:

  • Temperature (°C): Average temperature of the region.
  • Humidity (%): Relative humidity recorded in the region.
  • PM2.5 Concentration (µg/m³): Fine particulate matter levels.
  • PM10 Concentration (µg/m³): Coarse particulate matter levels.
  • NO2 Concentration (ppb): Nitrogen dioxide levels.
  • SO2 Concentration (ppb): Sulfur dioxide levels.
  • CO Concentration (ppm): Carbon monoxide levels.
  • Proximity to Industrial Areas (km): Distance to the nearest industrial zone.
  • Population Density (people/km²): Number of people per square kilometer in the region.

Target Variable:

  • Good: Clean air with low pollution levels.
  • Moderate: Acceptable air quality but with some pollutants present.
  • Poor: Noticeable pollution that may cause health issues for sensitive groups.
  • Hazardous: Highly polluted air posing serious health risks to the population.

Data Preprocessing

1. Feature Scaling (Standardization)

To ensure that all features contribute equally to the model, we scaled the features using StandardScaler. This scaling technique transforms the features to have a mean of 0 and a standard deviation of 1. This is particularly important for algorithms like Logistic Regression and Support Vector Machines (SVM), which are sensitive to the scale of input features.

2. Target Variable Encoding (Label Encoding)

The target variable, "Air Quality Levels", was encoded using Label Encoding. The original categories, such as "Good," "Moderate," and "Poor," were mapped to unique numeric labels:

  • Good -> Class 0
  • Moderate -> Class 1
  • Poor -> Class 2
  • Hazardous -> Class 3

Conclusion

The results of the air quality classification task demonstrate the effectiveness of various machine learning models in predicting the target variable with high accuracy. A comparative analysis of the models showed the following:

  • Random Forest: This model achieved the highest performance with an accuracy of 95.8%, precision of 99%, recall of 99%, and an F1-score of 99%. The ensemble nature of the Random Forest likely contributed to its robustness and ability to handle non-linear relationships effectively.
  • Logistic Regression: This model performed well, achieving an accuracy of 94%, but it was slightly outperformed by the Random Forest. While it is simpler and faster, it may not capture complex patterns as effectively.
  • Support Vector Machine (SVM): The SVM model also delivered strong performance with an accuracy of 94%, demonstrating its capability in handling the classification problem with a non-linear kernel.

Key Observations:

  • The Random Forest model outperformed others due to its ability to combine predictions from multiple decision trees, reducing overfitting and improving generalization.
  • The performance difference between Logistic Regression and SVM was minimal, showing both are competitive for this dataset.

Future Directions:

To further improve model performance and robustness, the following steps can be explored:

  • Incorporating additional features or external datasets, such as weather conditions or traffic data, to capture more variability.
  • Applying advanced hyperparameter tuning techniques, such as Bayesian optimization, for fine-grained model optimization.
  • Testing other ensemble methods, such as Gradient Boosting or XGBoost, which may yield even better results.

In conclusion, the Random Forest model demonstrated superior performance for this task, making it the preferred choice for predicting air quality levels based on the given dataset.
